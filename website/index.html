Intro

The Cloud Resume Challenge is designed to help aspiring engineers to gain practical knowledge within their respective cloud infrastructures. This challenge was created by cloud advocate and educator, Forrest Brazeal, who challenges all takers to showcase their skills through an online resume or portfolio.

The Cloud Resume Challenge involves creating and deploying a static website using online resources. I'll be using cloud technologies through Amazon Web Services (AWS) such as S3 buckets, CloudFront distributions, and Lambda functions to process data within a DynamoDB table.

I am excited to dive into this challenge, confident that it will not only enhance my technical skills but also provide valuable hands-on experience with cutting-edge cloud technologies. This endeavor aligns perfectly with my goal of continuous learning in the ever-evolving field of cloud computing.

Lessons

After achieving the AWS Certified Cloud Practitioner certification, I reached out to a Cloud Engineer at my workplace. They warmly welcomed me and, upon learning about my certification, recommended taking on the Cloud Resume Challenge to enhance my portfolio. This opportunity not only introduced a new challenge but also granted access to our environment's cloud sandbox, allowing me to work with AWS services without affecting our Production environment. Prior to taking on this challenge, my background was primarily theoretical, lacking significant hands-on experience in the cloud. However, I was undeterred by my limited cloud exposure and eagerly embraced the challenge.

First part: Certification and Skill Building

Unknowingly inaugurating the challenge, I secured the Foundational certification by passing the $100 exam on August 6th, 2023. Subsequently, I leveraged my knowledge from an introductory coding class in my Senior year of High School to undertake the hands-on task of crafting a resume using HTML and CSS. While I'm not a coding professional, I've utilized the example code provided in the challenge as a temporary placeholder while working through the 16 steps. I updated the code after verifying the successful execution of all other steps, which is reflected in the current version.

Second part: Deployment and Security

The next phase involves deploying the HTML resume as a static website using an Amazon S3 bucket. Following that, the resume is secured with HTTPS through Amazon CloudFront. This entails creating a CloudFront distribution linked to the S3 bucket and establishing a custom DNS domain directed to the CloudFront distribution

Hosting objects through an S3 bucket proved straightforward, but challenges arose when transitioning the static website to a secured CloudFront distribution. While linking the distribution to an object is simple, ensuring robust security requires additional steps. Opting for extra security, I created an S3 bucket policy, disabling the static website setting and enabling CloudFront access. This essentially limits access to the objects in my S3 bucket to the CloudFront distribution. The next hurdle involved creating a custom DNS domain and registering an SSL certificate. After overcoming multiple attempts and adjusting registration settings to DNS, I successfully registered my domain. Moving forward, the journey faced additional challenges in the realm of coding.

Third part: Development and Functionality

The third part emphasizes building a visitor counter using JavaScript, storing the count in a DynamoDB table. This involves crafting a Lambda function written in Python to manage and display the current visitor count on the static website.

This phase proved to be the most time-consuming, given my limited knowledge of Javascript, Python, and databases. Initially delving into Javascript through various YouTube tutorials, I faced a misdirection in the project. Following extensive self-study on Javascript, I then invested time in basic Python learning from Udemy courses. Despite consuming copious YouTube and Udemy content, my initial coding attempts ended in failure. It wasn't until I leveraged a ChatGPT account that I discovered critical logic issues, particularly related to case sensitivity in variables. After a weekend of debugging and refining, I successfully integrated a visitor counter into my JS code, along with a Lambda function in the correct Python language. This refined code executed the Lambda function, interacting with my DynamoDB table and updating the visit count on my website. Despite being the most challenging step, the satisfaction of seeing my website counter transition from displaying 'Website Counter' to the actual visit count made it immensely rewarding.

Fourth part: Continuous Integration and Deployment (CI/CD)

The fourth section addresses establishing GitHub repositories for both backend and website code, incorporating CI/CD via GitHub Actions for enhanced workflow efficiency. Ensuring security, this step emphasizes refraining from committing AWS credentials to source control.

While seemingly straightforward, this step not only provided valuable insights into GitHub repositories and Actions but also reshaped my understanding of code deployment, emphasizing the efficiency and security achieved through automation. It dispelled the notion of manual updates within developers' IDEs, highlighting the importance of streamlined processes in the workplace.

Fifth part: infrastructure Management

The last step involves defining resources in an AWS Serverless Application Model (SAM) template for Infrastructure as Code, and I've opted to execute this using Terraform.

My familiarity with this step was limited to hearing the name 'Terraform' in conversations with a Network engineer at work. Initially leaving it as my final step, despite being less proficient in VS Code (Terraform), I gained exposure to Terraform's functionalities for creating and deploying pipelines. As I bring the Cloud Resume Challenge to a close, I am inspired to continue my Terraform education, recognizing its significance in my workplace infrastructure. My goal is to elevate my skills and become a valuable asset to the Cloud team. Throughout this journey, having a Cloud Engineer mentor has been invaluable, providing guidance and steering me in the right direction at every obstacle. I look forward to applying the knowledge gained and contributing effectively to our cloud projects in the future.
